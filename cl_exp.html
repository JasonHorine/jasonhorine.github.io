CloudLogr

The CloudLogr site is the beginnings of a web-sever based data-logging product for the Industrial Automation market. The site is my MVP deliverable for an assignment to create and deploy my own application in about a week of coding. This assignment comprised the final week in the General Assembly Web Development Immersive course.

    About CloudLogr

  • Node.js is the runtime engine.
  • mLab hosts the MongoDB database.
  • CloudLogr is currently deployed on Heroku.

    How CloudLogr Works

  • Navigating to /tank initiates a route to send my credentials (stored as environment variables) to eWON's talk2m.com service and request a session ID.
  • From the JSON response, the session ID is picked out and then used to initiate a second request to retrieve data on a tank application available to me.
  • From the CSV response, the tank value is picked out and combined with a timestamp from the server and saved to the mLab database.
  • The response from mLab is used to dynamically build the table rendered to the user.
  • Refreshing the page repeats the process.

The above list represents what I was able to complete in just under a week. The list below is what I plan to do next.
